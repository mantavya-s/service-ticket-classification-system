# Wi-Fi or Eduroam Not Connecting

## Category
Network

## Subcategory
Wireless Connectivity

## Symptoms
Users may report that their computer or mobile device cannot connect to Wi-Fi or eduroam.

Common signs include:
- The network appears but the connection fails
- The device repeatedly asks for a username and password
- The device connects but has no internet access
- The user receives a certificate prompt
- The device previously connected but no longer works
- Other devices can connect in the same location

## Likely Causes
- The username or password is incorrect
- The saved wireless profile is corrupted
- The user's password recently changed
- A required certificate or configuration profile is missing
- The wireless adapter is disabled or malfunctioning
- The device date and time are incorrect
- The device is not using the required organizational username format
- The local access point is unavailable

## Troubleshooting Steps
1. Confirm whether the issue affects Wi-Fi generally or only eduroam.
2. Confirm airplane mode is off and Wi-Fi is enabled.
3. Confirm the correct network name is selected.
4. Verify the required organizational username format.
5. Ask whether the user recently changed their password.
6. Forget the wireless network and reconnect.
7. Restart the device.
8. Confirm the date, time, and time zone are correct.
9. Install the approved eduroam configuration profile or installer.
10. Do not accept an unexpected or unverified certificate prompt.
11. Test the device in another building or access-point area.
12. Test another device in the same location.
13. Update the wireless adapter driver if applicable.
14. Record any authentication or certificate error.

## Escalation Notes
Escalate if:
- Multiple users cannot connect in the same area
- The approved configuration profile fails
- Authentication logs show repeated failures
- The device cannot detect any wireless network
- A required certificate is missing
- The issue appears to be access-point related

Possible escalation team:
Network Operations / Identity and Access Management / Desktop Support

## Suggested Ticket Comment
I confirmed the affected wireless network, verified the username format, refreshed the saved Wi-Fi profile, checked the device date and time, and used the approved eduroam configuration method. I also tested whether the issue follows the user, device, or physical location.
The ticket will be escalated if wireless authentication, certificate deployment, or access-point troubleshooting is required.
