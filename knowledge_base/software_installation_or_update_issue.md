# Software Installation or Update Issue

## Category
Software

## Subcategory
Application Installation

## Symptoms
Users may report that an approved application will not install, update, or launch after installation.

Common signs include:
- The installer fails with an error
- The installation requires administrator credentials
- Company Portal or Software Center shows a failed installation
- The application remains on an old version
- The installation completes but the program does not open
- The device reports insufficient disk space

## Likely Causes
- The user does not have installation permissions
- The software is not approved or assigned to the user
- The device is low on storage
- An older version is blocking the installation
- The installer is damaged
- Windows requires a restart
- Endpoint security is blocking the installer
- The device is not connected to the corporate network or VPN

## Troubleshooting Steps
1. Confirm the software name, version, and business requirement.
2. Confirm the software is approved for organizational use.
3. Check whether the application is available through Company Portal or Software Center.
4. Confirm the device has sufficient free disk space.
5. Restart the computer.
6. Connect to the corporate network or VPN if required.
7. Remove a conflicting older version according to the approved procedure.
8. Retry the installation from the approved software source.
9. Record the exact installation error code.
10. Check installation logs when available.
11. Do not bypass security controls or use an unapproved installer.

## Escalation Notes
Escalate if:
- The application requires packaging or assignment
- Administrator elevation is required
- Endpoint security blocks an approved installer
- The installation fails on multiple managed computers
- A license must be purchased or assigned

Possible escalation team:
Desktop Support / Endpoint Management / Software Licensing
